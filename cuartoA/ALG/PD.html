1. Embarcaderos

A lo largo de un ro hay 7 embarcaderos conectados de la manera en que se muestra en la imagen. Se quiere obtener el recorrido ptimo entre el primer y el ltimo embarcadero, siendo este recorrido el de menor coste. Se harn varias implementaciones.

Aproximacin inicial

Se puede pensar que lo mejor sera llegar al embarcadero que menor coste tenga desde el embarcadero actual, pero esto no siempre consigue la mejor solucin al problema ya que hay rutas que no se consideran

Aproximacin recursiva

Se puede resolver este problema de manera correcta si lo resolvemos de forma recursiva mediante programacin dinmica. Primero plantearemos la ecuacin recursiva del problema.

Primero necesitamos determinar el espacio de soluciones del problema

Ser toda secuencia de embarcaderos que cumpla lo siguiente:

  • Empieza por 1 y acaba por el ltimo embarcadero E
  • Ninguno de los embarcaderos de la secuencia presenta saltos de ms de 2 embarcaderos.
$$ X = \{(e_1, e_2, \dots, e_n) \in [1..E]^+ \mid e_1 = 1; \quad e_n = E; \quad 1 \leq e_i - e_{i-1} \leq 2, \quad 1 < i \leq n\}. $$

A continuacin necesitamos definir una funcin objetivo que, en este caso minimizar para obtener la solucin deseada. Usaremos una funcin C que calcule el coste del camino entre 2 embarcaderos.

$$ (\hat{e}_1, \hat{e}_2, \dots, \hat{e}_n) = \arg \min_{(e_1, e_2, \dots, e_n) \in X} c((e_1, e_2, \dots, e_n)). $$$$ \min_{(e_1, e_2, \dots, e_n) \in X} c((e_1, e_2, \dots, e_n)). $$

Vemos que buscamos tanto el camino de menor coste como el menor coste asociado a dicho camino.

ECUACIN RECURSIVA

Necesitamos ahora buscar una expresin recursiva que defina el problema a resolver. Utilizaremos una expresin general que obtiene el coste mnimo posible entre 2 embarcaderos y demostraremos cmo se puede "desglosar" en llamadas recursivas de talla menor para obtener una solucin vlida al problema.

$$ \min_{\substack{e_1, e_2, ..., e_n \in X \\ 1 \leq i \leq n}} \sum_{i=1}^{n} c(e_{i-1}, e_i) = \min_{\substack{(e_1, e_2, ..., e_n), \\ 1 \leq i \leq n \\ e_1 = 1, e_n = E \\ 1 \leq e_i - e_{i-1} \leq 2}} \sum_{i=1}^{n} c(e_{i-1}, e_i) $$

Partiendo de esta expresin, y mediante la propiedad de los sumatorios en la que puedes extraer el ltimo elemento y sumrselo fuera del mismo, extenderemos la expresin a lo siguiente:

$$ \min_{\substack{e_1, e_2, ..., e_n \in X \\ 1 \leq i \leq n}} \sum_{i=1}^{n} c(e_{i-1}, e_i) = \min_{\substack{(e_1, e_2, ..., e_n-1), \\ 1 \le i \leq n-1 \\ e_1 = 1, 1 \leq E - e_{n-1} \leq 2 ; \\ 1 \leq e_i - e_{i-1} \leq 2}} ((\sum_{i=1}^{n-1} c(e_{i-1}, e_i) )+ c(e_{i-1}, E)) $$

Como sabemos que el penltimo embarcadero solo puede ser E - 1 o E - 2, podemos describir esta expresin de la manera siguiente:

$$ \min_{\substack{j \in [E-1, E-2]}} (\min_{\substack{(e_1, e_2, ..., e_n-1) | e_1 = 1; e_{n-1} = j; \\ 1 \leq e_i - e_{i-1} \leq 2 ; 1 \le i \leq n-1}} ((\sum_{i=1}^{n-1} c(e_{i-1}, e_i) )+ c(j, E))) $$

Utilizando ahora la propiedad del mnimo donde el mnimo de una suma a+b es igual a el mnimo de a + b... volvemos a reformular para obtener...

$$ \min_{\substack{j \in [E-1, E-2]}} ((\min_{\substack{(e_1, e_2, ..., e_n-1) | e_1 = 1; e_{n-1} = j; \\ 1 \leq e_i - e_{i-1} \leq 2 ; 1 \le i \leq n-1}} \sum_{i=1}^{n-1} c(e_{i-1}, e_i) )+ c(j, E)) $$

En la siguiente imagen podremos ver que tanto la parte izquierda de la igualdad como la parte interior de la derecha son muy similares. Solo se diferencian en que la parte izquierda es una expresin general para ir del embarcadero 1 al embarcadero E mientras que la parte derecha representa el coste hasta llegar al antepenltimo o ltimo embarcadero. Llegados a este punto podemos ver la recursin en la frmula.

Si abusamos de la notacin para definir una funcin C(j) que calcula el coste desde el primer embarcadero hasta el embarcadero j...

$$ C(j) = (\min_{\substack{(e_1, e_2, ..., e_n) | e_1 = 1; e_{n} = j; \\ 1 \leq e_i - e_{i-1} \leq 2 ; \; 1 \le i \leq n}} \sum_{i=1}^{n} c(e_{i-1}, e_i) ) $$

Podremos obtener el coste para C(E) de la manera siguiente...

$$ C(E) = \min \left\{ C(E-2) + c(E-2, E), \; C(E-1) + c(E-1, E) \right\} $$

El razonamiento para c(E) aplica tambin para cualquier otro embarcadero. Si queremos calcular el coste del camino ms barato entre el primer embarcadero y un embarcadero i cualquiera simplemente cambiamos esa E por una i.

Con esta frmula ya tenemos la base de nuestra recursin. Ahora slo necesitamos considerar nuestros casos base:

  • Si la recursin alcanza el embarcadero 1, hemos de devolver 0 ya que es un caso especial de embarcadero donde no hay ningn coste porque es desde el que se empieza.
  • Si la recursin alcanza el embarcadero 2 se debe aadir el coste que se obitene de viajar del embarcadero 1 al 2, ya que es el nico caso posible.
$$ C(i)= \begin{cases}0,\\c(1,2),\\min(C(i-2)+c(i-2,i),C(i-1)+c(i-1,i)),\end{cases} $$

Veamos ahora la implementacin de este mtodo:

In[6]:
def recursive_cheapest_price(E, c):
  def C(i):
    if i == 1: return 0
    elif i == 2: return c(1,2)
    else: return min(C(i-1)+c(i-1,i), C(i-2)+c(i-2,i))
  return C(E)

def memo_recursive_cheapest_price(E, c):
  R = {}
  def C(i):
    if i == 1: R[1] = 0
    elif i == 2: R[2] = c(1,2)
    else: 
      if i-2 not in R: C(i-2)
      if i-1 not in R: C(i-1)
      R[i] = min(R[i-1]+c(i-1,i), R[i-2]+c(i-2,i))
    return R[i]
  return C(E)

Aproximacin iterativa

El algoritmo recursivo paga un sobrecoste por llamadas recursivas. Es posible eliminarlo si hacemos nuestro problema iterativo. Con ello, hemos de analizar y resolver en orden las dependencias del problema, desde el embarcadero 1 hasta el i.

def iterative_cheapest_price(E, c):
  C = {}
  C[1] = 0
  C[2] = c(1,2)
  for i in range(3, E+1):
    C[i] = min(C[i-1] + c(i-1,i), C[i-2] + c(i-2,i))
  return C[E]

2. El problema de la mochila

El problema de la mochila busca maximizar el valor obtenido de meter una serie de objetos en una mochila con capacidad limitada. Cada objeto tiene un valor, cuya suma se busca optimizar, pero tambin tiene un volumen, por lo que no se pueden meter todos los objetos que se quieran en la mochila.

Aproximacin Recursiva

Hemos de buscar la solucin utilizando la programacin dinmica. Nos damos cuenta que el problema se puede expresar en subproblemas idnticos de talla menor que pueden ser resueltos de manera recursiva.

Primero averiguemos el set de soluciones factibles del problema.

Ser solucin todo aquel conjunto de objetos que cumpla las siguientes condiciones:

  • No debe exceder la capacidad total de la mochila

Dado un set de 7 objetos, se dice que un objeto entra en la mochila si el array booleano que representa dicho set tiene un 1 en la posicin que representa. Es decir, si hay 7 objetos, tendremos un array tal que [0,0,0,0,0,0,0]. En este caso, ningn objeto entra. si el array fuera [1,0,0,0,0,0,1], el primer y el ltimo objeto entraran.

$$ X = [{(x_1, x_2, ..., x_n) \in [1..N]^* \mid x_i \neq x_j, 1 \leq i \neq j \leq n; \sum_{1 \leq i \leq n} w_{x_i} \leq W }.] $$

De manera que buscaremos optimizar el argumento (en este caso sera el subset de objetos que metemos en la mochila) para el cual:

  • El peso no excede el de la mochila.
$$ X = [{(x_1, x_2, ..., x_N) \in {0,1}^N \mid \sum_{1 \leq i \leq N} x_i w_i \leq W }.] $$
  • Maximiza el valor.
$$ [f((x_{1},x_{2},...,x_{N}))=\sum_{1\le i\le N}x_{i}v_{i},] $$

De manera que la funcin objetivo que se busca maximizar es la siguiente:

$$ (\hat{x}_1, \hat{x}_2, ..., \hat{x}N) = \arg \max{(x_1, x_2, ..., x_N) \in X} f((x_1, x_2, ..., x_N)). $$

ECUACIN RECURSIVA

Para abordar el problema usaremos una estrategia top-to-down approach, en la que plantearemos la funcin recursiva desde el caso con el mayor beneficio posible a obtener. De tal manera...

$$ \max_{(x_{1},x_{2},...,x_{N})\in {0,1}^{N}\;\sum_{1\le i\le N-1}x_{i}w_{i}\le W}\sum_{1\le i\le N}x_{i}v_{i}=\max_{(x_{1},x_{2},...,x_{N})\in {0,1}^{N}\; \sum_{1\le i\le N-1}x_{i}w_{i}\le W}\left(\left(\sum_{1\le i\le N-1}x_{i}v_{i}\right)+x_{N}v_{N}\right). $$

La frmula es un sumatorio que busca el mximo valor para la suma de los valores de cada objeto seleccionado (con un 1 en la componente que le representa en el array booleano. Ntese que la definicin del set vlido de soluciones se define siempre en los argumentos de cada funcin max).

Se ha avanzado en la explicacin y se ha aplicado la propiedad de los sumatorios que permite separar el ltimo trmino del mismo y sumrselo al final restando 1 del lmite de iteracin.

Cuando decidimos si meter o no un objeto debemos tener en cuenta lo siguiente:

  • Hay espacio?
    • S:
      • Podemos meterlo y restarle capacidad a la mochila pero sumarle valor.
      • Podemos no meterlo y quedarnos con valor y capacidad inalteradas para cuando visitemos el objeto siguiente
    • No:
      • No lo metemos, no hay opcin.

Esta es la base de la recursin. A priori no sabemos si el objeto que metamos ahora mismo ser parte de la solucin ms ptima. Es por eso que debemos contemplar siempre los dos casos mencionados en caso de poder meterlo (en caso de no poder, no hay opcin. Esto tambin es un caso particular de la frmula de la recursin para este problema).

Para cada paso de la recursin buscaremos siempre el valor mximo entre el valor actual de nuestra mochila y el que tendra si le sumramos el objeto siendo "visitado" pero le restramos capacidad (posiblemente dejando por ende algunos objetos ms valiosos atrs, aunque el algoritmo recursivo asegura siempre la mejor solucin).

Apliquemos lo mencionado al elemento N-simo de los disponibles (esto nos conviene ya que vamos a ir del N-simo hasta el primero, por nuestra aproximacin top-down). Un uno significa que incluimos el objeto N-simo en la mochila, pero solo es posible si su peso es menor al de la capacidad total. Un cero indica que el objeto no se carga, cosa que puede hacerse siempre independientemente de la capacidad total de la mochila. Si el objeto se carga, los siguientes tendrn menos espacio para ser metidos, pero el valor del objeto actual se habr aadido. As pues...

$$ \max_{x_{N}\in \{0,1\}: x_{N}w_{N}\le W} \left( \max_{(x_{1},x_{2},...,x_{N-1})\in \{0,1\}^{N-1}: \sum_{1\le i\le N-1}x_{i}w_{i}\le W-x_{N}w_{N}} \left( \left( \sum_{1\le i\le N-1}x_{i}v_{i} \right) + x_{N}v_{N} \right) \right). $$

Ntese que la eleccin de meter o no el objeto s que est representada, ya que un valor de $ X_{N} $ igual a 0 elimina tanto el peso como el valor de la ecuacin.

Ya puede verse que en los parntesis interiores del problema se encuentra una instancia reducida del mismo que vuelve a calcular el valor del array de objetos hasta la posicin i. Abusando de la notacin podemos definir una funcin $ V(j, W) $ que encapsule este sumatorio y esta minimizacin para calcular el valor de los objetos hasta la posicin j limitados por un peso W, de manera que...

$$ V(N,W)=\max_{x_{N}\in {0,1}:x_{N}w_{N}\le W}\left(V(N-1,W-x_{N}w_{N})+x_{N}v_{N}\right) $$

Con esta frmula podemos establecer limpiamente nuestros casos de recursin de la siguiente manera:

$$ V(i,c) = \begin{cases} 0, & \text{si } i=0 \text{ o } c=0; \\ \max(V(i-1,c), V(i-1,c-w_i)+v_i), & \text{si } i>0, c>0, 0<w_i\le c; \\ V(i-1,c), & \text{si } i>0, c>0, 0<w_i>c. \end{cases} $$

Veamos ahora la implementacin de este mtodo:

def recursive_knapsack_profit(W, v, w):
  def V(i, c):
    if i==0 or c == 0: return 0
    elif c-w[i] >= 0: return max(V(i-1, c), V(i-1, c-w[i]) + v[i])
    else: return V(i-1, c)
  return V(len(v)-1, W)

3. Grafos

El problema del camino ms corto en un grafo acclico

Dado un grafo acclico ponderado G = (V, E, d), donde V son los vrtices, E son los edges (aristas), y d es la funcin que devuelve el coste entre dos vrtices... queremos encontrar el camino que parta del vrtice inicial y llegue al final con menor coste (lo que en el contexto del problema es sinnimo a menor distancia)

Sea...

$$ D(t) = min_{(v_1, v_2, ..., v_n) \in P(t)} \left( \sum_{1 \leq i \le n}d(v_i, v_{i+1}) \right) $$

La funcin que devuelve el camino de menor coste entre el vrtice inicial y el t, siendo P(t) el conjunto posible de caminos que cumplen ir del vrtice inicial (s) a t. (Ya que P indica los predecesores de su argumento)

Si vemos la ecuacin, podemos extraer trminos para dar con la recursin

$$ min_{(v_1, v_2, ..., v_n) \in P(t)} \left( \sum_{1 \leq i \le n}d(v_i, v_{i+1}) \right) = \left( \left( min_{(v_1, v_2, ..., v_{n}) \in P(t)} \sum_{1 \leq i \le n-1}d(v_i, v_{i+1}) \right) + d(v_n, v_{n+1}) \right) $$

Abusando de la notacin, podemos sustituir D(t) en el elemento interior de la ecuacin de la manera siguiente:

$$ min_{(v_1, v_2, ..., v_n) \in P(t)} \left( \sum_{1 \leq i \le n}d(v_i, v_{i+1}) \right) = \left( P(t) + d(v_n, v_{n+1}) \right) $$

Aproximacin recursiva

Damos entonces con el siguiente cdigo:

def iterative_shortest_acyclic_route(s, t, G, d):
  def D(v):
    if s == 0: return 0
    elif G.in_degree(v) == 0: return float('inf')
    else: return min(D(u) + d(u,v) for u in G.preds(v))

Ntese que asignamos infinito a los nodos sin predecesores para bloquear la expansin del algoritmo de minimizacin por ese camino.

Es posible averiguar el algoritmo iterativo de este problema entendiendo que en todo grafo acclico existe un orden topolgio de ordenacin para los vrtices, lo que permite recorrerlos en un sentido sin repeticiones. De tal manera, el orden topolgico del anterior grafo sera el siguiente

def iterative_shortest_acyclic_route(s, t, G, d):
  D = {}
  for v in topsort(G): # type: ignore
    if v == s: D[v] = 0
    elif G.in_degree(v) == 0: D[v] = float('inf')

El problema del camino ms corto en un grafo. Algoritmo de Bellman-Ford

El algoritmo de Bellman-Ford consigue el camino ms corto en un grafo sin importar que sea acclico siempre y cuando no tenga ciclos negativos. Esto lo hace partiendo del conocimiento de que, como mximo, el camino debe tener V-1 aristas de recorrido como mximo, ya que de contener ms tendra un ciclo (1 vrtice se visita ms de una vez), y el camino ptimo NUNCA contiene ciclos

def shortest_distance(G, d, s, t):
  D = dict(((v,0), float("inf") for v in G.V))
  D[s,0] = 0
  for i in range(0, len(G.V)):
    for v in G.V:
      if G.in_degree(v) == 0:
        D[v,i] = float("inf")
      else:
        D[v,i] = min(D[u, i-1] + d(u,v) for u in G.preds(v))
  return min(D[t,i] for i in range(len(G.V)))

4. Desglose de monedas

Se plantea el problema de dar el cambio de una cantidad con el menor nmero de unidades monetarias posible dado un sistema monetario.

  • Q = cantidad a devolver.
  • V = array que dicta el valor de cada tipo de moneda en el sistema monetario
  • x = array que almacena el nmero de monedas a devolver de cada tipo.

Siendo N la cantidad total de tipos de moneda distintos que hay en el sistema monetario... len(V) = N

1 Manera

Primero debemos considerar el espacio de soluciones factibles del problema...

$$ X = \{(x_1, x_2, x_3, ..., x_n) \in \mathbf{N} \mid \sum_{1 \leq i \leq n}{X_iV_i} = Q \} $$

De manera que una cadena solucin $ x $ estar contenida en el set de soluciones factibles $ X $

A partir de aqu podemos definir una ecuacin recursiva tal que...

$$ \min_{(x_1, x_2, x_3, ..., x_n) \in X \;\mid \;\sum_{1 \leq i \leq n}{X_iV_i} = Q}\left({\sum_{1 \leq i \leq n}{X_i}}\right) $$

Aplicamos la propiedad de los sumatorios de manera en que podamos obtener una definicin recursiva del problema (encontrar el subproblema). Hemos adelantado la expansin de la frmula para expresar correctamente la recursin, de manera que se busca el mnimo para los valores de $ X_n $ menores al suelo de la divisin entre el valor actual de Q y el valor de N procesado en una instancia del problema. Esto se hace para determinar la cantidad adecuada de monedas del tipo N (de entre las posibles a seleccionar entre 0 y Q/Vn que minimizara la cantidad de monedas a devolver).

$$ \min_{(x_1, x_2, x_3, ..., x_n) \in X\; \mid \;\sum_{1 \leq i \leq n}{X_iV_i} = Q} \left({\sum_{1 \leq i \leq n}{X_i}}\right) = \min_{0 \leq X_n \leq \lfloor{Q/V_{n}}\rfloor} \left(\min_{(x_1, x_2, x_3, ..., x_{n-1}) \in X\; \mid \;\sum_{1 \leq i \leq n-1}{X_iV_i} = Q-X_nV_n} \left({\sum_{1 \leq i \leq n-1}{X_i}}\right) + {X_n} \right) $$

Vemos claramente que podramos definir una funcin auxiliar que contuviese la parte interior de la frmula de manera que devolviera la cantidad mnima de monedas para un valor Q usando hasta n tipos de moneda (de 0 a n). Definiramos esta funcin de la manera siguiente...

$$ F(Q,n) = \min_{0 \leq X_n \leq \lfloor{Q/V_{n}}\rfloor} \left(\min_{(x_1, x_2, x_3, ..., x_{n-1}) \in X\; \mid \;\sum_{1 \leq i \leq n-1}{X_iV_i} = Q-X_nV_n} \left( {\sum_{1 \leq i \leq n-1}{X_i}} \right) \right) $$$$ F(Q,n) = \min_{0 \leq X_n \leq \lfloor{Q/V_{n}}\rfloor} \left( F(Q-X_nV_n, n-1) + X_n \right) $$

En cdigo, sera lo siguiente:

Aproximacin recursiva

def recursive_money(V, Q):
  N = len(V)
  def F(Q,n):
    if Q == 0: return 0
    elif n == 0: return float('inf')
    else: return min(F(Q - i*V[n], n-1)+ i for i in range(0,Q/V[n]))
  return F(Q, N)

def iterative_money(V, Q):
  R = dict(((0,q), float("inf") for q in range(0, Q)))
  R[0,0] = 0
  for v in range(1, len(V)):
